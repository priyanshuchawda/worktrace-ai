# Release Channels

Status: active policy, 2026-05-27.

WorkTrace AI uses local/internal builds while product development continues. The
repository may publish source-only engineering milestones, but it must not
publish unsigned Windows installer binaries as public GitHub Release assets.

## Channels

| Channel | Purpose | Distribution | Updates |
| --- | --- | --- | --- |
| `dev` | Local development and validation | Local worktree builds only; NSIS may be built for internal QA | No auto-update |
| `alpha` | Engineering/source milestone | GitHub draft/prerelease notes and source only; no unsigned installable Windows binaries | No auto-update |
| `store-beta` | Future Microsoft Store beta | Deferred Store-compatible MSIX/AppX submission path | Expected to be Store-managed |
| `store-stable` | Future public stable channel | Deferred until Store beta evidence and owner approval | Expected to be Store-managed |

## Versioning

- Use `v0.x.y-alpha.n` for source-only engineering milestones.
- Use `v0.x.y-store-beta.n` only after Store-compatible packaging and
  submission/certification evidence exists.
- Use `v1.0.0` only after trusted distribution, privacy proof, CI, and real
  user-value proof are complete.

## GitHub Releases

GitHub Releases are allowed for release notes, source milestones, and safe
aggregate validation summaries. They are not a public Windows installer
distribution channel for the current product phase.

Allowed alpha release contents:

- release notes
- source archive generated by GitHub
- safe validation summaries without local private paths
- links to documentation and issue/PR milestones

Forbidden alpha release contents:

- unsigned `.exe`, `.msi`, `.msix`, `.msixbundle`, `.appx`, `.appxbundle`
- installer/update `.zip` bundles
- Tauri updater manifests or artifacts
- API keys, `.env` values, model files, screenshots, OCR text, raw reports, or
  user session evidence

## Artifact Guard

Before creating an engineering release from a local artifact directory, run:

```powershell
pwsh -File scripts/release/validate-release-artifacts.ps1 -ArtifactPath <path> -Channel alpha
```

The guard fails closed when installable or update-like artifacts are present.
It is intentionally conservative: alpha releases should be source/release-notes
only.

Future Store channels may allow installable artifacts only after the owner has
approved Store packaging work and the release operator sets an explicit approval
environment variable. That approval is not used in the current alpha path.

## Update Policy

The Tauri updater remains disabled for public distribution. Future Store
channels should use Microsoft Store-managed updates unless the owner later
approves and verifies a separate trusted direct-distribution channel.

No update manifest may contain API keys, local paths, prompt bodies, raw
evidence, user machine details, model files, or screenshot content.
